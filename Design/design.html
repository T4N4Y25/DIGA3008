The IxD Process

The IxD Process was looked at in the week 4 blog Post and general ideas were put together for this.

Goal Alignment

There are three major factors to look at when deciding on goal alignment for the website: who is the target audience? What should the users see and do? What is the most important and interesting content.

  • The Target Audience:
    There are two main target audiences for the website, the lecturers and markers for the course as well as potential employers in the game design and software engineering fields.
  • What should the users see and do:
    Considering the target audience the website should be presented professionally. The user will be presented with options to navigate different sections of the website which will each showcase a different aspect of myself or my work. The main pages to be included are a general homepage with an overview of the site and navigation options, a design section detailing the research and design process for creating the site, a blog post page where the user can find the weekly blogs done for the course, an essays page showcasing the two essays which will be done later in the course, a profile page explaining to the user who I am, a portfolio page showcasing some of the projects and work I have completed, and finally I am also including a section which will have a copy of my CV. The interactions for the site will mostly be for navigation purposes as the website is primarily to showcase myself.
  • The most important and interesting content
    Considering the users will be the lecturers and markers, the main content to be displayed should be the work done for the course, hence the essays, blog posts, and design section. Potential employers will also be looking at the site so the projects page will also be important to display, and this will most likely be the most interesting section of the website.

User Flow

The user flow is designed so the user can access any page from all the main page sections indicated by arrows. Each block represents a screen for the website. The only exception is not allowing the user to directly access the other pages via the individual weekly blog posts and the individual project pages. This is to avoid to many links on these pages which can lead to confusion for the user as these pages will be information heavy and it is better to have a single link back to the main respective page. Main pages will however all have access to a common navigation bar and all pages will allow the user to return to the homepage.

Interface Elements

The main interface elements will be buttons, links, scrollbars, control interfaces and progress bars.

  • Buttons:
    Buttons will be included on each weekly blog to allow the user to chronologically navigate the blogs. On the essays page a button will allow the user to seamlessly switch between the two essays which will be displayed on the main essay page.
  • Links:
    Links will be the primary source of navigation using the general navigation bar. The user will be able to navigate to each section of the website using links on the homepage. Links will be included to allow the user to view a specific project on its own respective page via the portfolio page.
  • Scrollbars:
    Scrollbars will be included on each page excluding the homepage and the blogs home page. This will allow the user to easily navigate through information heavy pages and will allow more components (images, media clips etc.) while retaining their ideal size in contrast to compressing the components to fit on the page.
  • Control interfaces:
    For the video game projects to be displayed in the portfolio section of the website, a WebGL media player can be implemented to allow the user to directly try out the video game being displayed.
  • Progress bars:
    This will allow the user to track how far they are in pages such as the weekly blogs and the projects. It will give the user a good idea of how much more content the pages have to offer.

Updates to IxD Process

The inclusion of JavaScript allows for updates within the IxD process. One key update is the Information Structure as JavaScript allows more control as to what can be seen at a time.

The user flow remains relatively the same with the exception of the projects displayed which will no longer be displayed via their own page but rather through a filtering system which can be incorporated through JavaScript. The interface elements also remain the same with a primary focus on links, buttons, and progress bars.

I want the general style of the website to mimic the aesthetic of a vintage nighttime city skyline best represented by 90s anime cartoons and movies. The colour choices should therefore match this style and should be either a mix of blues and black or purples and black. Style Guide

The main goal and aesthetic are to invoke a 90s anime-style city theme to the website.

This would mean having a primarily dark look to the website contrasted heavily by bright text and icons like a city at night where the light comes from buildings, signs etc.

The fonts I initially wanted to use were not available on Google Fonts. I therefore researched more fonts that would match the city-pop feel of the website.

The visual design elements I choose need to incorporate the feeling of a retro anime-style city. I have chosen several design choices to help encompass this aesthetic:

  • Neon lines for borders around cards that take the user to a certain page (making these borders flicker can add to the aesthetic).
  • Scanlines over the website (commonly seen in 90s anime and VHS tapes) for a nostalgic feeling.
  • Simple icons for the navigation bar to each relevant page. Hover-over text will explain what each icon represents.
  • Buttons will look like signs therefore square with a background colour contrasting the main text colour. They will also appear elevated with a shadow behind them to create a 3D effect. The font will flicker between white and the main neon blue/purple colour when a cursor hovers over the button to indicate interactivity.
  • Building images will represent the page cards that can take the user to the various pages.
  • A logo representing the website in the Navigation Bar (logo will be in the main colour of the site – Neon Blue).

The hierarchy of the website will be as follows:

  • The title of the website will have the largest font size (H1: 50px).
  • Subsequent headings and titles will have equal font sizes (H2: 33px).
  • Body text will have the smallest font size (18px).
  • All paragraph text will be left-aligned for easier reading. Icons and visual elements will be centre aligned.

Updates to the Style Guide

The style guide held up quite well with only a few minor changes needed such as a change to the background colour (changed from black to a transparent navy blue - #050514F2) as the black background contrasted to heavily with the white text and caused eye-strain.

The homepage style also changed due to the ability to create effects with JavaScript. Instead of a gif that plays as the background for the homepage, a rain effect will be created using JavaScript to keep the background colour consistent across all pages while still pertaining to the late-night city theme.

The navigation bar will no longer consist of icons acting as the links as these icons are not intuitive to users. Instead simple text will act as the links allowing for the user to know where they are going at a first glance.

The two fonts used - Audiowide and Barlow both work very well together and are consistent with the theme. Audiowide can become illegible if used in paragraphs so this font will be reserved specifically for main headings and card titles. Barlow is a legible font and will be used for the paragraph text, however the text being white can become illegible when appearing in a paragraph so a gap of around 2px-5px will be implemented to help make the text more legible.

To help contain paragraphs within sections such as essays and blogs, a thin, white border will be placed around the text to negate empty space created by margins. Text will be left-aligned but the main body of text will be centred to prevent the user from needing to read from one end of the screen to the other.

Wireframes

Homepage:

The luminous purple borders around each image will flicker, similar to a neon sign turning on, when the user hovers over that specific image to add more interactivity and feedback for the user. This will also make the page card the user is hovering over stand out more using the Similarity and Difference effect which can help entice users to visit the page.

Blogs Homepage:

Blogs:

The Next blog and Previous blog button may be moved to the sides of the progress bar as these two interface elements along with the progress bar can help implement the Serial Position Effect (last items are remembered). The progress bar provides a good opportunity for implementing the UX design principle where uncompleted tasks are remembered. This will allow the user to better identify their progress through the blog readings and will promote them to continue reading through the subsequent blogs. The colour white (#ffffff) is predominantly used on the blog pages due to the large quantity of reading done and this colour can reduce eye strain compared to the brighter and neon colours.

Portfolio page:

The portfolio section acts as a “shop” in a sense. The purpose of this section is to “sell” your skills by demonstrating your projects; hence the title of each section will have a flickering logo to simulate the appearance of a shop sign late at night.

The left-hand section is reserved for quick access to help simplify choices within the UX. Instead of scrolling through everything the user now has the option to quickly navigate to the project they want to specifically view.

The colour scheme for the quick links was altered to be more consistent with how each colour has been used (blue is important text and headings, white is general text, purple is borders and yellow is to indicate an interactable interface element such as links and buttons).

Design page:

The design page is similarly designed to the portfolio page where the quick links will allow the user to quickly navigate to where they want to view.

About page:

The profile section should be short and simple while giving users enough information about me. Included will be an image depicting myself, some general information on me, my contact details (choosing email as well as GitHub for now) and an option to view or download my CV.

An update has been made to the homepage design. The animated gif for the background has been removed to keep the background colour consistent across all pages. The page cards are also navigated through scrolling with the mouse or swiping with a touchscreen to traverse the page cards. An indicator in the footer will help give clarity on the number of pages the user can scroll through.
